Lottery Prediction System - Software Documentation & Architecture

üìã System Overview

BayanWin is a modern, full-stack web application that predicts lottery numbers using multiple machine learning models. The system scrapes historical lottery data from Google Sheets, stores it in InstantDB (a Backend-as-a-Service platform), and uses 5 different ML algorithms to generate predictions for multiple lottery games.

Key Capabilities: The system supports 5 lottery games (Ultra Lotto 6/58, Grand Lotto 6/55, Super Lotto 6/49, Mega Lotto 6/45, and Lotto 6/42) and provides real-time predictions, statistical analysis, error distance tracking, and Gaussian distribution visualization.

ü§ñ Machine Learning Models

BayanWin employs 5 distinct ML models, each using different algorithms and approaches to generate predictions:

1. XGBoost Model

Type: Gradient Boosting

Approach: Uses historical patterns and gradient boosting trees to identify complex relationships in lottery number sequences.

Performance: ~6-10 seconds per prediction

2. Decision Tree Model

Type: Random Forest Classifier

Approach: Frequency-based analysis using ensemble of decision trees to predict likely numbers based on historical frequency.

Performance: ~4-6 seconds per prediction

3. Markov Chain Model

Type: State Transition Model

Approach: Models number sequences as state transitions, predicting next numbers based on transition probabilities from historical data.

Performance: ~1-3 seconds per prediction

4. Normal Distribution Model

Type: Gaussian Distribution Analysis

Approach: Analyzes sum and product distributions of winning numbers, selecting numbers that fit high-probability Gaussian patterns.

Performance: ~0.1-0.5 seconds per prediction

5. Deep Reinforcement Learning (DRL)

Type: Reinforcement Learning Agent

Approach: Uses deep neural networks with 3 feedback loops. Learns from accuracy records to optimize predictions over time (5 episodes per prediction).

Performance: ~20-40 seconds per prediction

Special Feature: Continuously improves through feedback loop from accuracy calculations

‚öôÔ∏è Technical Implementation Details

Backend Architecture

The FastAPI backend serves as the central orchestrator, handling all business logic, ML model coordination, and data operations. It uses:

  • Async/Await: Non-blocking operations for better performance
  • CORS Middleware: Enables frontend-backend communication
  • Pydantic Models: Request/response validation
  • Concurrent Processing: ML models run with timeout protection
  • Background Tasks: Auto-calculation of accuracy runs asynchronously

Frontend Architecture

The React frontend provides a modern, responsive user interface:

  • Component-Based: Modular React components for maintainability
  • State Management: React hooks for local state management
  • Real-time Updates: Immediate display of predictions as models complete
  • Error Handling: Graceful error states and user feedback
  • Visualizations: Charts and graphs for statistical analysis

Database Schema

InstantDB stores three main entity types:

  • Results: Historical lottery draw results (draw_date, numbers, jackpot, winners)
  • Predictions: ML model predictions (model_type, predicted_numbers, target_draw_date)
  • Accuracy Records: Prediction accuracy metrics (error_distance, numbers_matched, distance_metrics)
